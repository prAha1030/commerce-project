# 커머스 과제

실시간 커머스 플랫폼

## 플랫폼 구조

- Main
- Product
- CommerceSystem
- Category
- Customer

## 플랫폼 상세 내용

**Main**

- 상품의 정보를 생성하여 각각의 카테고리에 분류
- 플랫폼 시스템 작동

**Product**

- 상품명, 가격, 설명, 재고, 장바구니에 담긴 수량을 지닌 상품

**CommerceSystem**

- 메인 페이지
    - 1: 전자제품
    - 2: 의류
    - 3: 식품
    - 4: 장바구니 확인
    - 5: 장바구니 비우기 (주문 취소)
    - 6: 관리자 모드
    - 0: 종료


- 메인 1~3번 카테고리 선택
    - 1: 전체 상품, 2: 100만원 이하 상품, 3: 100만원 이상 상품
    - 상품 선택 시 장바구니 추가
- 메인 4~5번
    - 4: 장바구니 내역 확인 및 주문
        - 고객 등급 선택에 따른 할인율 적용
    - 5: 장바구니에 담긴 상품 모두 제거
- 메인 6번 관리자 모드 (비밀번호 3회 실패 시 메인으로)
    - 1: 상품 추가
        - 카테고리 선택
        - 상품명, 가격, 설명, 재고 설정
    - 2: 상품 수정
        - 상품명 검색
        - 해당 상품 수정 (1: 가격, 2: 설명, 3: 재고)
    - 3: 상품 삭제
        - 카테고리 선택
        - 상품 선택 시 삭제
    - 4: 전체 상품 현황

**Category**

- 상품들을 저장하는 카테고리

**Customer**

- 이름, 이메일, 등급을 지닌 고객
- 등급에 따른 할인율
    - BRONZE: 0%
    - SILVER: 5%
    - GOLD: 10%
    - PLATINUM: 15%

---

## 주의사항

번호, 가격, 재고 입력칸
-> 양의 정수 이외의 값 or 문자 입력 시 오류 발생
-> 프로그램 종료
